# ModParks 最小要件定義 (MVP)

## 目的

Minecraft Java Edition向けのMod・Pluginを簡単に公開・検索・ダウンロードできるサービスを提供する。

* Plugin

# ModParks 今後の優先開発ロードマップ (Roadmap)

実装済みの機能を踏まえ、残りの最大要件定義を優先度順のフェーズに分けて整理しました。

## フェーズ3: ユーザー体験とコミュニティの拡大（高優先度）

ユーザーが定着しやすく、より多くのコンテンツを扱えるようにするフェーズです。

### チュートリアル・案内機能
- 登録後にウェブサイトの使い方を案内するオンボーディング

### 対応コンテンツの拡張
- Resource Pack
- Data Pack
- Shader
- Modpack
- World
- Addon
- Script
- Tool
- Minecraft Bedrock Edition 対応

### コミュニティ機能
- コメント・レビュー
- ユーザー/プロジェクトのフォロー
- お気に入り機能
- 開発日記（Devlog）
- Q&Aセクション

---

## フェーズ4: 開発者体験 (DX) の向上（中優先度）

開発者が ModParks をメインのプラットフォームとして選びやすくなる仕組みを構築します。

### Git連携・リリース自動同期
- GitHub連携 / GitLab連携
- GitHub Release の自動同期（リリース時に自動で ModParks の Version を作成）

### 開発者支援ツール
- ダウンロード統計ダッシュボード
- 利用者分析
- CI/CD連携
- Webhookの提供

---

## フェーズ5: セキュリティとAIによる高度化

プラットフォームの安全性を担保し、AIでユーザーサポートを強化します。

### セキュリティ強化
- ウイルススキャン / マルウェアチェック
- ファイルハッシュ検証
- 署名検証
- 著作権侵害の自動審査

### AI機能
- **翻訳**: 説明文の自動多言語翻訳（中国語、韓国語、欧州言語などへの対応）
- **タグ生成**: プロジェクト内容からタグの自動提案
- **カテゴリ判定**: 分類の自動提案
- **要約**: 長文説明文の自動要約
- **不正検知**: スパムや不適切コンテンツの自動判定

---

## フェーズ6: プラットフォームの独立と収益化（長期目標）

### ModParks Launcher（専用ランチャー）
- ランチャー内でのMod検索
- ワンクリックインストール・アップデート機能
- Modpack管理機能

### 収益化モデル
- プラットフォーム内の広告
- 開発者への寄付機能
- サブスクリプション・スポンサーシップ

---

## 最終目標

Minecraft向け日本発サービスとして、
CurseForge と Modrinth の代替ではなく、

**「日本人開発者ならまずModParksに出す」**

という状態を目指す。
