# ModParks (Core Platform)

ModParksは、日本発のMinecraft Java Edition向けMOD、プラグイン・リソースパック等の公開・共有プラットフォームです。
ユーザーは自身が作成したプロジェクトを投稿し、バージョンごとにファイルを管理・配布することができます。

## プラットフォームの目的と特徴

Minecraftのコミュニティにおいて、MODやプラグインの配布は複数の海外プラットフォームに分散していますが、日本のユーザーにとっては言語の壁やサポートの面でハードルがある場合があります。ModParksは、**直感的でモダンなUI**と**多言語対応（日本語・英語）**を提供し、誰でも簡単にプロジェクトを公開・検索できるハブとなることを目指しています。

### 主な機能

1. **プロジェクト管理**
   - MOD、プラグイン、リソースパック、データパックなどのプロジェクトを作成可能。
   - バージョン管理システム: 対応するMinecraftバージョン（1.20.1など）やローダー（Fabric, Forge, NeoForge, Paperなど）を指定してファイルをアップロード。
   - Markdown形式でのリッチなプロジェクト説明文（README）の記述。

2. **充実した検索・フィルター機能**
   - プラットフォーム（MOD / プラグイン）、対応バージョン、ローダーによる高度な絞り込み検索。
   - ダウンロード数や更新日時による並び替え。

3. **コレクション・リスト機能**
   - ユーザーがお気に入りのプロジェクトをまとめた「リスト（コレクション）」を作成・公開可能。
   - 他のユーザーが作成したリストをフォローして最新のMOD構成を追跡。

4. **コミュニティ機能（アイデア・フィードバック）**
   - 「アイデア」としてユーザーが欲しいMODやプラグインの構想を投稿し、開発者と交流できる場を提供。
   - 開発者と利用者が直接コミュニケーションを取れるコメント・フィードバック機能。

5. **外部連携・統合**
   - CurseForge や Modrinth などの主要プラットフォームと連携。
   - 外部プラットフォームからのダウンロード数の合算表示、同期機能。
   - **GitHub Webhook連携**: GitHubのReleaseと連動し、新しいバージョンを全自動でModParksに取り込む機能を搭載。

## 技術スタック・アーキテクチャ

ModParksは、モダンなWeb技術をベースに構築されたフルスタックアプリケーションです。

- **フロントエンド / バックエンド**: Next.js 15 (App Router)
- **UIライブラリ**: Material-UI (MUI v5)
- **言語**: TypeScript (厳格な型チェックを適用)
- **データベース ORM**: Drizzle ORM
- **データベース**: Cloudflare D1 / Turso (SQLiteベース)
- **ファイルストレージ**: Cloudflare R2 (S3互換ストレージ)
- **認証**: NextAuth.js (GitHub, DiscordなどのOAuth対応)
- **多言語対応 (i18n)**: next-intl (汎用テキストは `Common` 名前空間等で一元管理)

## プロジェクトディレクトリ構成

主要なディレクトリ構成は以下の通りです：

- `app/` : Next.js App Routerのルーティングとページ定義（`[locale]` で多言語対応）
- `components/` : 再利用可能なUIコンポーネント（機能ごとにディレクトリ分割）
- `db/` : Drizzle ORMのスキーマ定義とマイグレーションファイル
- `lib/` : ユーティリティ関数、APIクライアント、サーバーアクション等のビジネスロジック
- `messages/` : 多言語対応用のJSONファイル (`en_us.json`, `ja_jp.json`)
- `docs-md/` : プラットフォーム全体のドキュメント・仕様書
